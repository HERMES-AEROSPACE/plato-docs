Module Details

1. driver

  • plato_initialize() – main entry point, loads all data files

  • plato_finalize() – cleanup

2. general

  • Stores runtime data: number of species, components, etc.

  • Global physical constants (\(k_B\), etc.)

3. interface, interface_c, interface_cpp

Three separate directories provide the external API:

  • ``interface/`` – Primary Fortran API (plato_library_interface.F90)

  • ``interface_c/`` – C API wrapper via iso_c_binding (plato_library_interface_c.c)

  • ``interface_cpp/`` – C++ API wrapper (plato_library_interface_cpp.cxx)

Headers for the C/C++ APIs live in the top-level include/ directory (15 .h files).

Key design points:

  • The Fortran interface is a pure re-export facade – no executable code, only interface blocks that rename internal routines with the plato_ prefix.

  • The C and C++ wrappers independently call the same Fortran bind(C) functions. The C++ layer does NOT wrap the C layer.

  • For routines needing C interoperability, PLATO provides dual versions: an assumed-shape dimension(:) version (Fortran-facing, F-suffixed) and an assumed-size dimension(*) version (C-facing, with bind(C)).

4. post_process

The post-processing module (3 source files) extracts derived quantities and internal temperatures from CR/StS population distributions. Depends only on thermo.

  • get_output_var(nb_in_var, nb_out_var, in_var, out_var) [C-bound] – converts primitive variables to derived quantities for visualization

  • post_process_solution(...) – Newton solver for Boltzmann-equivalent internal temperatures (warm-starting, REL_TOL = 1e-8, MAX_IT = 50)

5. utils

  • File parsing (database text files)

  • String manipulation

  • Interpolation routines

  • Curve fitting: modified Arrhenius law

Source Term Pipeline

  1. Convert mass densities \(\rho_i\) to number densities \(n_i\)

  2. Clamp temperatures (enforce positivity)

  3. compute_rate_coeff() – forward via modified Arrhenius, backward via microreversibility

  4. Loop over 14 reaction categories, accumulate production rates: \(\dot{n} = k_f \times [\text{reactants}] - k_b \times [\text{products}]\)

  5. Convert to mass rates: \(\omega_i = m_i \times \dot{n}_i\)

  6. Add energy exchange: VT, VE, CV, CEL, TE, FF

Analytical Jacobian

Full \(\partial(\omega_i, \Omega) / \partial(\rho_j, T_k)\) matrix with:

  • Chain rule for Park temperature

  • Collision partner optimization via jcol temporary

  • Transfer module Jacobians included

  • Mass-only variant: \(n_{sp} \times n_{sp}\)

8. ode_integrator

ODE right-hand sides for simplified configurations:

  • BOX: isochoric reactor (isothermal or adiabatic)

  • SHOCK: inviscid post-shock flow

  • LAGR: Lagrangian inviscid flow

  • GSI: gas-surface interaction [new]

Time integration via LSODE (legacy, Fortran 77) or CVODE from SUNDIALS.

9. thermo

The largest module (~28 source files) providing two alternative thermodynamics paths:

Path 1: Partition-function-based (default)

  • Statistical mechanics: rigid rotor, harmonic oscillator, Boltzmann electronic

  • Equilibrium constants from partition function ratios

  • No entropy available

Path 2: NASA 9-coefficient polynomials (--enable-nasa_poly)

  • nasa_thermo_class type with 51 type-bound procedures across 7 submodules

  • Provides Cp, h, s, g (Gibbs) per species

  • Entropy available (absolute entropy from polynomials)

11. kinetics_gsi – Gas-Surface Interaction [new]

8 surface reaction types: catalysis (gamma), adsorption, desorption, dissociative chemisorption, Langmuir-Hinshelwood (2 variants), Eley-Rideal (2 variants).

Three species namespaces: gas / surface / bulk. All forward-only (no microreversibility). QSS Newton solver for steady-state surface coverages.

12. kinetics_lpi – Laser-Plasma Interaction [new]

Multi-photon ionization (MPI) and inverse Bremsstrahlung (IB) source terms with analytical Jacobian. Absorption coefficient delegated to radiation module. All processes irreversible.

13. thermo_gsi – GSI Thermodynamics [new]

Surface/bulk species data for gas-surface interaction (2 files). Query routines for surface species names, counts, and gas-phase mapping.

14. radiation – Radiative Properties [new]

Spectral absorption/emission coefficients (9 source files). OOP composition pattern: radiation type contains rad_atom(:) and rad_molecule(:) arrays.

Processes:

  • Bound-Bound (BB): Line-by-line with Voigt profiles, NIST Einstein coefficients

  • Bound-Free (BF): Kramers hydrogenic cross-section

  • Free-Free (FF): Electron-ion Coulomb + electron-neutral tabulated Qff

LTE mode computes equilibrium composition internally; NLTE mode accepts user-supplied populations.

Limitation: Molecular BB/BF not implemented – only FF for molecules.

15. proj_rom – Projected Reduced-Order Models [new]

Petrov-Galerkin projection with biorthogonalized trial (\(\Phi\)) and test (\(\Psi\)) bases:

  • Encode: \(z = \Psi^T \times (x - x_{ref})\)

  • Decode: \(x = \Phi \times z + x_{ref}\)

  • Jacobian compression: \(J_{red} = \Psi^T \times J \times \Phi\)

Mask array allows mixing projected and non-projected species.
